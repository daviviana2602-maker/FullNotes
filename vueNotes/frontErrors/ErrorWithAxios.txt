==================================
    handleApiError com Axios
==================================


export function handleApiError(error: unknown) {

}

    Centraliza o tratamento de erros da API.
    Sempre que uma Service faz uma requisição usando o Axios e ela falha (400, 401, 404, 500...), o Axios cria um AxiosError e lança esse erro para o catch.



if (!axios.isAxiosError(error) || !error.response)

    Verifica se o erro veio de uma requisição feita por uma Service usando o Axios.
    Se não for um AxiosError, significa que o erro não veio da requisição HTTP da API, então retorna uma mensagem genérica.



const apiError = error.response.data as ApiError;
    Pega o JSON retornado pelo backend e converte para a interface ApiError.


Exemplo:

{
    "errorCode": "EMAIL_ALREADY_EXISTS",
    "message": "Email already exists"
}



return ErrorMessages[apiError.errorCode] ?? "Ocorreu um erro inesperado.";

    Procura a mensagem correspondente ao errorCode no ErrorMessages:


               EMAIL_ALREADY_EXISTS

                        ↓

            "Este e-mail já está em uso."


Se o backend retornar um errorCode que não existe no ErrorMessages, retorna:

    "Ocorreu um erro inesperado."
